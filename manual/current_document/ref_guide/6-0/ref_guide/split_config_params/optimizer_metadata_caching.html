optimizer_metadata_caching

启用GPORCA（默认值）时，此参数指定GPORCA在查询优化期间是否在inHybrid数据库master上的内存中缓存查询元数据（优化数据）。 此参数的默认值为on，启用缓存。 缓存是基于会话的。 会话结束时，缓存将被释放。 如果查询元数据的数量超过缓存大小，则从缓存中逐出旧的未使用的元数据。

如果该值为off，则GPORCA在查询优化期间不会缓存元数据。

可以为数据库系统，单个数据库或会话或查询设置此参数。

服务器配置参数optimizer_mdcache_size控制查询元数据缓存的大小。
