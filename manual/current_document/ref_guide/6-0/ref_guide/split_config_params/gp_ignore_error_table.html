gp_ignore_error_table (不推荐使用)

在CREATE EXTERNAL TABLE或COPY命令中指定不推荐使用的INTO ERROR TABLE子句时，控制inHybrid数据库行为。

默认值为false，如果在命令中指定了INTO ERROR TABLE子句，inHybrid数据库将返回错误。

如果值为true，则inHybrid数据库会忽略该子句，发出警告，并在没有INTO ERROR TABLE子句的情况下执行该命令。 在inHybrid数据库中，您可以使用内置SQL函数访问错误日志信息。 请参见 CREATE EXTERNAL TABLE或COPY命令。

您可以将此值设置为true， 以避免在运行包含inHybrid数据库 INTO ERROR TABLE子句的CREATE EXTERNAL TABLE或COPY命令的应用程序时出现inHybrid数据库错误。

在inHybrid数据库5中不推荐使用INTO ERROR TABLE子句并将其删除。 在inHybrid数据库7中，此参数也将被删除，导致所有INTO ERROR TABLE调用产生语法错误。
