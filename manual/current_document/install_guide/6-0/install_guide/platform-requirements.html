操作系统

inHybrid 6在以下操作系统平台上运行：

  • Red Hat Enterprise Linux 64-bit 7.x见以下注释
  • Red Hat Enterprise Linux 64-bit 6.x
  • CentOS 64-bit 7.x
  • CentOS 64-bit 6.x
  • Ubuntu 18.04 LTS
重要提示：在RedHat 6.x和CentOS 6.x系统上启用基于资源组的工作负载管理时，发现inHybrid数据库性能显著下降。此问题是由Linux cgroup内核错误引起的。此内核错误已在CentOS 7.x和Red Hat 7.x系统中修复。

注意：对于7之前安装在Red Hat Enterprise Linux 7.3之前或CentOS 7.3之前上的inHybrid数据库，操作系统问题可能会导致运行大量工作负载的inHybrid数据库挂起工作负载。inHybrid数据库问题是由Linux内核错误引起的。RHEL7.3和CentOS7.3解决了这个问题

inHybrid数据库服务器在RHEL/CentOS系统上支持TLS版本1.2，在Ubuntu系统上支持TLS版本1.3

硬件和网络

下表列出了在Linux系统上支持inHybrid数据库的硬件服务器的最低推荐规范。inHybrid数据库系统中的所有主机服务器必须具有相同的硬件和软件配置。
警告：在超聚合基础设施（HCI）上运行inHybrid数据库存在性能、可伸缩性和稳定性方面的已知问题，不建议这样做。
表1.最低硬件要求
最小CPU 任何x86_64兼容的CPU
最小内存 每台服务器16 GB内存
磁盘空间要求
  • 每个主机150MB，用于inHybrid安装
  • 元数据的每个段实例大约300MB
  • 磁盘容量不超过70%的数据有适当的可用空间
网络要求 阵列中的10千兆以太网

当存在多个接口时，建议使用NIC绑定

inspur inHybrid可以使用IPV4或IPV6协议。

存储

您应该在XFS文件系统上运行inHybrid数据库。

如果将共享存储作为块设备提供给运行inHybrid数据库的服务器，并且XFS文件系统安装在块设备上，inHybrid数据库可以在网络上运行，也可以在共享存储上运行。不建议使用网络文件系统。使用网络或共享存储时，inHybrid数据库镜像的使用方式必须与本地存储相同，并且不应修改镜像方案或段的恢复方案。

共享存储的其他功能，如重复数据消除和/或复制，只要不影响inHybrid数据库的预期操作，就可以与inHybrid数据库一起使用。

inHybrid数据库只能部署到虚拟化系统中，前提是存储以块设备的形式显示，并且XFS文件系统是为segment存储目录而装载的。
警告：在超聚合基础设施（HCI）上运行inHybrid数据库存在性能、可伸缩性和稳定性方面的已知问题，不建议将其作为关键inHybrid数据库的可伸缩解决方案，如果稳定性问题与基础设施相关，则可能不受关键inHybrid数据库的支持。HCI虚拟化了传统硬件系统的所有元素，至少包括虚拟计算、虚拟SAN和虚拟网络。

inHybrid数据库可以使用Amazon实例存储（Amazon使用卷名ephemeral[0-20]）或Amazon弹性块存储（Amazon EBS）在Amazon Web服务（AWS）服务器上运行。如果使用Amazon EBS存储，那么存储应该是amazonebs卷的RAID，并与XFS文件系统一起装载。

Hadoop分布式

inHybrid数据库通过inHybrid平台扩展框架（PXF）提供对HDF的访问。PXFv5.10.0与inHybrid Database 6集成，提供对Hadoop、对象存储和SQL外部数据存储的访问。有关PXF配置和使用信息，请参阅inHybrid数据库管理员指南中的使用PXF访问外部数据。

PXF可以使用Cloudera、Hortonworks数据平台、MapR和通用Apache Hadoop发行版。PXF捆绑了它所依赖的所有JAR文件，包括以下Hadoop库：

表二.PXF Hadoop支持的平台
PXF版本 Hadoop 版本 Hive Server 版本 HBase Server 版本
5.10.0 2.x, 3.1+ 1.x, 2.x, 3.1+ 1.3.2
5.8.2 2.x 1.x 1.3.2
5.8.1 2.x 1.x 1.3.2
注意：如果您计划访问存储在Cloudera Hadoop集群中的JSON格式数据，PXF需要Cloudera 5.8版或更高版本的Hadoop发行版。
