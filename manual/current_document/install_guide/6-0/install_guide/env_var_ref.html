必需的环境变量

注意:可以通过从inHybrid数据库安装目录中获取greenplum_path.sh文件来设置HDBHOME、PATH和LD_LIBRARY_PATH
Parent topic: inHybrid 环境变量

HDBHOME

这是inHybrid数据库软件的安装位置。例如:

HDBHOME=/usr/local/greenplum-db-6.x.x
export HDBHOME

PATH

您的路径环境变量应该指向inHybrid Database bin目录的位置。例如:

PATH=$HDBHOME/bin:$PATH
export PATH

LD_LIBRARY_PATH

LD_LIBRARY_PATH环境变量应该指向inHybrid数据库/PostgreSQL库文件的位置。例如:

LD_LIBRARY_PATH=$HDBHOME/lib
export LD_LIBRARY_PATH

MASTER_DATA_DIRECTORY

这应该指向hdbinitsystem工具在主数据目录位置中创建的目录。例如:

MASTER_DATA_DIRECTORY=/data/master/gpseg-1
export MASTER_DATA_DIRECTORY

可选环境变量

下面是标准的PostgreSQL环境变量，在inHybrid数据库中也可以识别。为了方便起见，您可能希望将与连接相关的环境变量添加到配置文件中，这样您就不必在命令行上为客户机连接键入那么多选项。注意，这些环境变量应该只在inHybrid数据库Master主机上设置。

Parent topic: inHybrid 环境变量

PGAPPNAME

应用程序的名称，通常由连接到服务器的应用程序设置。此名称显示在活动视图和日志条目中。PGAPPNAME环境变量的行为与application_name连接参数相同。application_name的默认值是psql。名称不能超过63个字符。

PGDATABASE

连接时使用的默认数据库的名称。

PGHOST

inHybrid数据库主主机名。

PGHOSTADDR

Master主机的IP地址。可以将其设置为PGHOST或PGHOST之外，以避免DNS查找开销。

PGPASSWORD

如果服务器要求口令身份验证，则使用的口令。出于安全原因，不建议使用此环境变量(一些操作系统允许非根用户通过ps查看流程环境变量)。相反，可以考虑使用~/.pgpass文件。

PGPASSFILE

用于查找的密码文件的名称。如果没有设置，它的默认值是~/.pgpass。有关更多信息，请参阅PostgreSQL文档中关于密码文件的主题。

PGOPTIONS

为inHybrid数据库主服务器设置其他配置参数。

PGPORT

Master主机上inHybrid数据库服务器的端口号。默认端口是5432。

PGUSER

用于连接的inHybrid数据库用户名。

PGDATESTYLE

设置会话的日期/时间表示的默认样式。(相当于将datestyle设为…)

PGTZ

设置会话的默认时区。(相当于将时区设置为…)

PGCLIENTENCODING

设置会话的默认客户端字符集编码。(相当于将client_encoding设置为…)
