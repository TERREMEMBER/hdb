hdbdeletesystem

hdbdeletesystem

删除一个使用hdbinitsystem初始化的inHybrid数据库系统hdbinitsystem。

概要

hdbdeletesystem [-d master_data_directory] [-B parallel_processes]  
                [-f] [-l logfile_directory] [-D]
                 
                hdbdeletesystem -?  
                 
                hdbdeletesystem -v

描述

hdbdeletesystem工具执行以下两种操作：

  • 停止所有postgres进程 （Segment实例和Master实例）。
  • 删除所有的数据目录。

在运行hdbdeletesystem之前：

  • 将所有备份文件移出Master数据目录和Segment数据目录。
  • 确保inHybrid数据库在运行。
  • 如果用户当前位于Segment数据目录中，请将目录更改为另一个位置。 从Segment数据目录中运行时，该工具会失败，并显示错误。

该工具不会卸载inHybrid数据库软件。

选项

-d master_data_directory
指定Master主机数据目录。如果未指定此选项，则使用环境变量MASTER_DATA_DIRECTORY的设置。 如果指定了此选项，则会覆盖MASTER_DATA_DIRECTORY的任何设置。如果无法确定 master_data_directory，则该工具返回错误。
-B parallel_processes
并行删除的Segment数。如果未指定，则该工具将根据需要删除多少个Segment实例启动 最多60个并行进程。
-f (force)
即使在数据目录中找到备份文件，也强制删除。如果备份文件存在，默认不删除inHybrid 数据库实例。
-l logfile_directory
写入日志文件的目录。默认为~/hdbAdminLogs。
-D（调试）
设置日志级别为debug。
-?（帮助）
显示在线帮助。
-v（显示工具版本）
显示该工具的版本、状态、上次更新的日期和校验和。

示例

删除一个inHybrid数据库系统：

hdbdeletesystem -d /hdbdata/hdb-1

删除一个inHybrid数据库系统，即使备份文件存在：

hdbdeletesystem -d /hdbdata/hdb-1 -f
