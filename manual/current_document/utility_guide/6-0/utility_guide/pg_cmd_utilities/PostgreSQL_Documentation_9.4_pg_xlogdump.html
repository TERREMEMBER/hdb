pg_xlogdump

名字

pg_xlogdump -- 显示PostgreSQL数据库集群的写前日志的可读呈现

概要

pg_xlogdump [option...] [startseg [endseg] ]

描述

pg_xlogdump 显示写前日志(WAL)，主要用于调试或教育目的。

这个工具只能由安装服务的用户运行，因为它要求对数据目录进行只读访问。

选项

以下命令行选项控制输出的位置和格式:

startseg

从指定的日志segment文件开始读取。这将隐式地确定搜索文件的路径和要使用的时间线。

endseg

读取指定的日志segment文件后停止。

-b
--bkp-details

输出关于备份块的详细信息。

-e end
--end=end

在指定的日志位置停止读取，而不是读取到日志流的末尾。

-f
--follow

在到达有效的WAL结束后，保持每秒轮询一次，以便出现新的WAL。

-n limit
--limit=limit

显示指定数量的记录，然后停止。

-p path
--path=path

要在其中查找日志segment文件的目录。默认情况下是在当前目录的pg_xlog子目录中搜索它们。

-r rmgr
--rmgr=rmgr

仅显示由指定的资源管理器生成的记录。如果将list作为名称传递，请打印有效资源管理器名称的列表，然后退出。

-s start
--start=start

开始读取的日志位置。默认情况下是开始读取在找到的最早文件中找到的第一个有效日志记录。

-t timeline
--timeline=timeline

从其中读取日志记录的时间线。默认值是使用startseg中的值(如果指定了该值);否则，默认值为1。

-V
--version

打印pg xlogdump版本并退出。

-x xid
--xid=xid

只显示标有给定TransactionId的记录。

-?
--help

显示关于pg_xlogdump命令行参数的帮助，并退出。

注释

在服务器运行时可能给出错误的结果。

只显示指定的时间线(如果没有指定，则显示默认时间线)。其他时间线中的记录将被忽略。
